# 타입 변환(Type Conversion) 실무 적용 사례

Go 언어의 타입 변환은 실무 백엔드 개발(API 서버 등)에서 매일 쓰이는 핵심 기술입니다.
가장 빈번하게 발생하는 3가지 주요 실무 시나리오를 정리합니다.

---

## 1. 웹(URL) 데이터 처리 (`strconv.Atoi`)
웹 브라우저나 앱에서 백엔드로 보내는 데이터는 모두 **문자열(String)**로 취급됩니다.

* **실무 상황**: URL에서 페이지 번호나 상품 ID를 읽어올 때
* **예시 URL**: `https://api.shop.com/products?page=2`
* **적용**: 
  ```go
  pageNumber, err := strconv.Atoi("2")
  ```
* **이유**: 클라이언트가 보낸 `"2"`를 실제 정수 `2`로 변환해야만 DB 쿼리에 사용할 수 있습니다.

## 2. 고유 키 생성 및 로깅 (`strconv.Itoa`)
반대로 백엔드의 숫자 데이터를 Redis 같은 캐시 서버나 로그에 저장할 때는 문자열로 합쳐야 합니다.

* **실무 상황**: 특정 사용자의 장바구니 데이터를 저장할 캐시 키를 만들 때
* **적용**: 
  ```go
  userId := 104
  cacheKey := "cart_user_" + strconv.Itoa(userId) 
  // 결과: "cart_user_104"
  ```

## 3. 통계 및 정산 계산 (`float64()`, `int()`)
DB에서 가져온 데이터(주로 `int`)를 바탕으로 평균, 할인율 등을 계산할 때 필수적입니다.

* **실무 상황**: 총 결제액을 결제자 수로 나누어 1인당 평균 결제액을 구할 때
* **적용**: 
  ```go
  totalSales := 100500
  userCount := 3
  
  // 정확한 소수점 계산을 위해 float64로 변환 후 연산
  avg := float64(totalSales) / float64(userCount)
  ```
* **이유**: `int` 상태로 나누면 소수점이 강제로 버려져 정산 시 심각한 금액 오차가 발생할 수 있습니다.
