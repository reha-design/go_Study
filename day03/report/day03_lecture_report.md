# Go 강의 보고서

## 1. 강의 목표 및 성취 수준

본 강의는 Go 언어의 제어 흐름을 주도하는 조건 제어문(if, switch)과 조건식의 복합성을 높여주는 논리 연산자, 그리고 반복 작업을 자동화하는 단일 루프 제어문(for) 및 중첩 반복문의 원리를 학습하고, 각 개념을 직접 코드로 작성·실행해 보는 것을 목표로 진행하였습니다.

수강생들은 강의와 실습을 통해 다음 내용을 수행할 수 있도록 학습하였습니다.

1. if, else if, else 조건문의 기초적인 구조를 체득하고 프로그램의 갈림길을 설계할 수 있다.
2. Go 언어만의 고유하고 효율적인 if 초기화 문법(if 초기문; 조건식)을 사용하여 변수의 생명주기(Scope)를 엄격하게 다스릴 수 있다.
3. 논리 연산자(&&, ||, !)의 동작과 진리표 기준을 분석하여 여러 조건을 논리적으로 매끄럽게 연결할 수 있다.
4. switch문의 값 매칭 규칙을 학습하고, 조건절을 생략한 분기 방식 및 의도적으로 다음 case를 자동 수행시키는 fallthrough 키워드의 활용법을 적용할 수 있다.
5. Go에 존재하지 않는 while문 대신 단 하나의 반복 키워드인 for문을 활용하여 일반 반복, 조건식 반복, 무한 루프를 유연하게 빌드할 수 있다.
6. 반복 동작의 흐름을 통제하는 break(완전 탈출)와 continue(이번 회차 건너뛰기)의 기능적 경계를 이해하고 제어할 수 있다.
7. 이중 반복문(중첩 루프)을 설계하여 구구단 표와 같은 2차원 공간의 수열 연산을 구조적으로 설계하고 순차 탐색할 수 있다.

#### ① `12_if_conditional/main.go`

```go
package main

import "fmt"

func main() {
	// [1. 기본 if문]
	// 조건이 참(true)일 때만 블록 내부의 코드를 실행합니다.
	score := 85

	fmt.Println("--- 1. 기본 if-else 구조 ---")
	if score >= 90 {
		fmt.Println("합격 등급: A")
	} else if score >= 80 {
		// 바로 앞 조건이 거짓이고, 현재 조건이 참일 때 실행됩니다.
		fmt.Println("합격 등급: B")
	} else {
		// 위의 모든 조건이 거짓일 때 최종 실행됩니다.
		fmt.Println("합격 등급: C")
	}

	// [2. if 초기화 문법]
	// Go 언어에서는 if 조건식 직전에 짧은 변수 선언 및 초기화를 수행할 수 있습니다.
	// 이 변수는 해당 if-else 블록 내부에서만 살아있고 블록이 끝나면 소멸합니다. (메모리 절약에 효과적)
	fmt.Println("\n--- 2. if 초기화 문법 ---")
	if tempScore := 95; tempScore >= 90 {
		fmt.Printf("임시 점수 %d는 A등급입니다.\n", tempScore)
	}
	// 이곳에서 tempScore 변수를 출력하려고 하면 컴파일 에러가 발생합니다.
	// fmt.Println(tempScore) // 에러!
}
```

#### ② `13_logical_operators/main.go`

```go
package main

import "fmt"

func main() {
	// [논리 연산자: 조건들을 엮어서 더 스마트한 판단하기]
	// 비교 연산의 결과들을 묶어 종합적인 참/거짓을 판별합니다.

	age := 25
	hasLicense := true

	fmt.Println("--- 1. 논리 AND (&&) ---")
	// && : "그리고" (양쪽 조건이 모두 참이어야만 최종 참이 됨)
	if age >= 20 && hasLicense {
		fmt.Println("운전이 가능합니다.")
	} else {
		fmt.Println("운전이 불가능합니다.")
	}

	fmt.Println("\n--- 2. 논리 OR (||) ---")
	// || : "또는" (두 조건 중 단 하나라도 참이면 최종 참이 됨)
	isWeekend := true
	isHoliday := false
	if isWeekend || isHoliday {
		fmt.Println("학교나 회사에 가지 않고 쉽니다!")
	} else {
		fmt.Println("일과를 진행합니다.")
	}

	fmt.Println("\n--- 3. 논리 NOT (!) ---")
	// ! : "아님" (참을 거짓으로, 거짓을 참으로 뒤집음)
	isRainy := false
	if !isRainy {
		fmt.Println("날씨가 맑으니 우산이 필요 없습니다.")
	}
}
```

#### ③ `14_switch_conditional/main.go`

```go
package main

import "fmt"

func main() {
	// [switch문: 값이 딱딱 맞아떨어지는 분기 처리]
	// if-else if 구조가 너무 길어질 때 사용하면 가독성이 획기적으로 올라갑니다.
	rank := 2

	fmt.Println("--- 1. 기본 switch문 ---")
	switch rank {
	case 1:
		fmt.Println("금메달입니다!")
	case 2:
		fmt.Println("은메달입니다!")
	case 3:
		fmt.Println("동메달입니다!")
	default:
		// 매칭되는 case가 없을 때 최종 실행됩니다.
		fmt.Println("아쉽게도 메달 획득에 실패했습니다.")
	}

	// [2. 다중 조건 case 및 조건식이 생략된 switch문]
	// case 뒤에 쉼표(,)를 사용해 여러 매칭 조건을 나열할 수 있습니다.
	// switch 뒤에 대상을 생략하면 내부 case 조건식을 if문처럼 유연하게 활용할 수 있습니다.
	fmt.Println("\n--- 2. 다중 조건 및 조건식 생략 switch문 ---")
	score := 88
	switch {
	case score >= 90:
		fmt.Println("성적 평가: 우수")
	case score >= 70 && score < 90:
		fmt.Println("성적 평가: 보통")
	default:
		fmt.Println("성적 평가: 노력 요함")
	}

	// [3. fallthrough 키워드]
	// Go 언어의 switch문은 기본적으로 일치하는 case가 실행되면 자동으로 종료됩니다. (break 자동 적용)
	// 만약 다음 case의 동작까지 강제로 이어서 실행하고 싶다면 fallthrough 키워드를 사용합니다.
	fmt.Println("\n--- 3. fallthrough 키워드 활용 ---")
	num := 1
	switch num {
	case 1:
		fmt.Println("case 1 실행")
		fallthrough // 바로 아래 case 2도 조건 검사 없이 강제로 들어갑니다.
	case 2:
		fmt.Println("case 2 실행 (fallthrough 효과)")
	case 3:
		fmt.Println("case 3 실행")
	}
}
```

#### ④ `15_for_loop/main.go`

```go
package main

import "fmt"

func main() {
	// [for문: 컴퓨터가 제일 잘하는 반복 작업 자동화]
	// Go 언어에는 다른 언어에 있는 while문이 없으며, 오직 'for' 키워드 하나로 모든 반복을 해결합니다.

	// [1. 기본 for문]
	// for 초기문; 조건식; 증감식 { ... }
	fmt.Println("--- 1. 기본 1부터 5까지 반복 ---")
	for i := 1; i <= 5; i++ {
		fmt.Printf("%d ", i)
	}
	fmt.Println()

	// [2. 조건식만 있는 for문 (while 형태)]
	// 초기문과 증감식을 생략하고 조건만 남겨 다른 언어의 while문처럼 동작하게 합니다.
	fmt.Println("\n--- 2. 조건만 있는 반복문 ---")
	count := 3
	for count > 0 {
		fmt.Printf("카운트다운: %d\n", count)
		count--
	}

	// [3. break와 continue 제어]
	// break: 반복문을 그 자리에서 완전히 폭파(종료)하고 탈출합니다.
	// continue: 아래 코드를 무시하고 다음 바퀴(반복 주기)로 즉시 넘어갑니다.
	fmt.Println("\n--- 3. break & continue ---")
	fmt.Println("   [1부터 10 중 짝수만 출력하되, 8을 만나면 탈출]")
	for i := 1; i <= 10; i++ {
		if i%2 != 0 {
			continue // 홀수면 아래 출력 건너뛰고 다음 바퀴로 점프
		}
		if i == 8 {
			fmt.Println(" 8을 만났으니 중단합니다!")
			break // 반복 완전 탈출
		}
		fmt.Printf("%d ", i)
	}
	fmt.Println()
}
```

#### ⑤ `16_nested_loop/main.go`

```go
package main

import "fmt"

func main() {
	// [중첩 반복문: 반복문 안에 또 반복문 집어넣기]
	// 2차원적인 표(Grid) 형식의 데이터를 다루거나 순서쌍을 연산할 때 자주 활용됩니다.
	// 가장 대표적인 실습 예제인 '구구단'을 작성해 봅니다.

	fmt.Println("--- 중첩 반복문 실습: 구구단 출력 ---")

	// 바깥쪽 for문: 2단부터 9단까지 단수를 제어합니다.
	for dan := 2; dan <= 9; dan++ {
		fmt.Printf("=== %d단 ===\n", dan)

		// 안쪽 for문: 각 단마다 1부터 9까지 곱해지는 숫자를 제어합니다.
		for i := 1; i <= 9; i++ {
			fmt.Printf("  %d * %d = %d\n", dan, i, dan*i)
		}
		fmt.Println() // 단마다 구분 빈 줄 출력
	}
}
```

## 3. 핵심 교육 내용 요약

### 조건 제어와 변수의 스코프 제한

가장 전형적인 양방향 분기문인 `if-else` 구조의 동작 흐름을 파악하고 실습을 수행했습니다. 이어서 Go 언어의 고유한 구조 설계인 `if 초기문; 조건식` 문법을 학습하였습니다.

해당 형식을 활용하면 조건 검증 로직에만 일시적으로 관여하는 전용 변수(예: 호출 성공 여부, 임시 연산 결과)를 선언하여 조건문 블록 내부에서 유연하게 통제할 수 있습니다. 조건 판단이 끝나 `if-else` 영역을 완전히 벗어날 경우 컴퓨터 메모리에서 해당 식별자가 소거되므로 네이밍 혼선 및 불필요한 메모리 점유를 조기에 차단해 프로그램의 안전성을 배가시킨다는 강력한 혜택을 이해하였습니다.

### 논리곱, 논리합, 논리부정의 조합

비교 연산의 동시 만족이나 우회 검증 처리를 위해 3가지 종류의 논리 연산자 사용법을 심층적으로 확인하였습니다.

- 논리 AND (`&&`): 결합된 조건들이 동시에 모두 `true`를 충족할 때 최종 참으로 귀결
- 논리 OR (`||`): 연동된 판단 조건들 중 단 하나만 성립해도 전체 흐름을 참으로 통과
- 논리 NOT (`!`): 불리언 참/거짓 판단 결과를 역으로 번복하여 참은 거짓으로, 거짓은 참으로 변전시킴

이를 통해 게임 계정의 접속 인가 처리(성인 판정 AND 본인 인증 여부 확인 등)처럼 현실 세계의 복합적인 업무 규칙들을 로직 코드로 그대로 이식하는 방법을 실습하였습니다.

### switch문 분기 및 fallthrough 활용법

값의 다중 일치 판정을 가독성 높게 코디네이팅하는 `switch`문을 정밀 실습하였습니다. Go 언어의 `switch` 제어문은 타 개발 언어와 변별되는 고유한 편의 및 강제적 요소를 보유하고 있음을 인지하였습니다.

- 개별 case에 break 키워드 자동 내장: 일치하는 선택 조건을 찾아 해당 실행 구절이 소화되면, 별도의 지시 없이도 switch 블록을 자동으로 완전 이탈 처리함.
- case 그룹 다중 나열: case 구절 뒤에 쉼표 `,`를 병렬 배치하여 1개 상자 안에 여럿의 허용 값을 동시에 바인딩 지원.
- switch 식별 대상 생략: 대상을 지정하지 않고 `switch`를 공백 처리하면 case 절에 직접 수식을 명문화하여 사실상의 직관적 `if-else` 분기 처럼 구획화 가능.
- 명시적 `fallthrough` 도입: 기본으로 작동하는 break 탈출 기능을 정지시키고, 바로 직하단의 case 블록 구절까지 조건 만족 여부 판단을 완벽히 무시한 채 강제로 수행을 밀고 나가는 연계 기법.

### 단일 반복문 for 제어와 분기 탈출

컴퓨터의 가장 강력한 재능인 수열 연산 및 무한 복사 반복 처리를 구동하기 위한 유일무이한 수단으로 `for` 루프를 관찰하였습니다.
Go 언어 개발자들은 `while`이나 `do-while`처럼 용도가 겹치고 복잡한 별도의 키워드를 과감히 정리하고 오직 `for`라는 핵심 단어 하나로 모든 반복 구조를 통일 설계했음을 학습했습니다.

- 표준 순환: `for 초기문; 조건식; 증감식`으로 고전적인 회차 반복 수행
- 조건 순환: 초기문과 증감식을 삭제하고 `for 조건식` 형태만 기재해 while식 제어로 변형
- 탈출 제어문 `break`: 무한 회전이나 연속 동작 중 목표 임계 영역에 다다랐을 때 루프를 파괴하고 바로 다음 로직으로 탈출시킴
- 탈출 제어문 `continue`: 잔여 실행 코드를 단절하고 반복 제어의 증감식 혹은 조건 판별부로 즉각 턴을 강제 스킵시킴

### 중첩 루프 구조와 2차원 영역 제어

순차 루프 내부에 또 다른 독립 순환 루프를 탑재하여 다차원 공간의 원소들을 격자(Grid) 구조로 제어하는 중첩 for문을 작성하였습니다.
중첩 루프의 구동 시 바깥쪽 제어 변수가 1회 변경될 때마다 안쪽 제어 변수가 전체 주기를 완전 순회하게 되는 시간적 흐름 관계를 실증 예시인 '구구단(2단~9단 단수 제어 및 1~9 승수 연산)' 출력을 통해 명료하게 분석하였습니다.
격자 테이블 정합 및 2D 픽셀 탐색, 복합 정렬 등의 고급 알고리즘 설계를 향한 소중한 기본기를 완성하였습니다.

## 4. 강의 평가 및 교육적 성과

이번 3일차 강의에서는 프로그램에 지능적인 갈림길을 설계해주는 다중 조건 제어문(if, switch)과 조건 연산식의 유연성을 증폭해주는 논리 연산자, 그리고 반복 프로세스를 효율적으로 운용하는 단일 for문 및 중첩 for문의 기초적 뼈대를 탄탄히 수립하였습니다.

수강생들은 if문 안에서 선언된 변수가 블록 밖으로 누수되지 않도록 관리하는 Go 특유의 if 초기화 변수의 수명 스코프 이론을 습득하여 구조적인 메모리 절약법을 파악하였습니다. 또한, 헷갈리기 쉬운 switch문 내 break 자동 적용 규칙과 fallthrough의 연속 강제 동작 관계를 상호 교차 대조하여 실수 없는 스위칭 분기 연출력을 체득하였습니다.

가장 위대한 교육적 도약은 다양한 루프 제어 기법을 단 하나의 `for` 키워드로 치환 운용하는 Go의 간결한 언어 철학을 직접 코드 기법으로 소화하고, 루프를 스킵하는 `continue`와 폭파하는 `break`를 논리 조건식에 알맞게 배열함으로써 정밀한 순환 흐름을 내재화했다는 점입니다.

종합해보면, 수강생들은 Go 언어의 제어 및 순환에 대한 모든 코어 원리를 매끄럽게 통달하였으며, 4일차에 다룰 연속성 대용량 메모리 저장 구조이자 실무 개발의 중추인 **배열(Array)과 슬라이스(Slice), 그리고 데이터 맵(Map)** 을 능숙하게 정렬하고 탐색할 만반의 준비 태세를 완벽히 성취하였습니다.
